# System E Kernel

A typechecker kernel for a proof system based on the formal system **E** (Euclid’s Elements). The file `proof.txt` contains an example proof.

- Dependent type theory (no inductive types)
- All the built-in types and rules from System E
(like Point and Circle)

## Build the project and run tests
First, install OCaml and opam and then create a switch for the project.
```bash
opam init
opam switch create nicegeo 5.4.1
eval $(opam env --switch=nicegeo)
```

Install the dependency:
```bash
opam install --deps-only --with-test .
```

Build the project:
```bash
dune build
```

Run the kernel on the test file:
```bash
dune exec nicegeo proof.txt
```

## Running Tests

```bash
dune runtest
```

## Project Structure
```
nicegeo/
├── dune-project              # Top-level dune configuration and package metadata
├── tarski-kernel.opam        # Opam package file (auto-generated by dune)
├── README.md                 # This file
├── .gitignore
├── bin/
│   ├── dune                  # Declares the main executable
│   └── main.ml               # Entry point that checks if the proof is correct
├── kernel/                   # Type checking logic
│   ├── dune                  # Declares the main executable
│   └── infer.ml              # defines the `inferType` and `isDefEq` functions.
│   └── term.ml               # Defines the basic kernel-level `term` types.
├── elab/                     # Usability feature layer that handles parsing, filling in holes and producing kernel-level term  
│   ├── elab.ml               # Contains the main interface of the layer
│   ├── typecheck.ml          # Contains type checking and unification 
│   ├── lexer.mll             # Defines the grammar for the parser
│   └── parser.mly            # Defines the parser 
```
